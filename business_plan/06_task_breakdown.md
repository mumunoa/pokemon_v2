# 06. 実行タスク分解 (Task Breakdown)

3ヶ月でリリースし、月10万円を達成するための月次マイルストーンとタスクです。

## [Month 1] コア体験の開発とα版リリース
**目標**: 圧倒的に快適なUIを持つ、サンドボックス一人回しツールの完成
- [x] UI/UXデザインのFigma詳細化（ポケポケ風ステータス・PTCGL風レイアウト）。
- [x] Vercel + Next.js (React) でのベースプロジェクト立ち上げ。
- [ ] 公式デッキコード（文字列）を入力し、画像・カードリストを取得するスクレイパーAPIの開発。
- [/] Zustand / Redux等を用いた「GameState」の設計（カードの位置、裏表、HP、各種ステータス）。
- [/] ドラッグ＆ドロップ、タップ等のタッチインターフェースの実装。
- [ ] テストユーザー（競技プレイヤー層5〜10名）へのクローズド展開と操作感のヒアリング。

## [Month 2] アカウントシステム・付加価値機能の実装とβ版公開
**目標**: ログインによる状態保存と、独自の分析機能の追加、一般公開
- [ ] Supabase / Clerk を用いたユーザー認証機能の実装。
- [ ] サンドボックスの「盤面保存＆復元機能（Save / Load）」の実装。
- [ ] 確率計算・ドローシミュレーション（分析機能）の実装。
- [ ] X（Twitter）での盤面画像生成・シェア機能の実装（OGP画像動的生成など）。
- [ ] X（Twitter）上でのβ版プロモーション開始。「スマホでポケカを一人回しできる神ツール完成しました」の投稿。

## [Month 3] マネタイズ基盤の実装とグロース
**目標**: サブスク機能の解放、プロモーション強化と10万の収益達成
- [ ] Stripeを用いた月額課金（300円/月）連携の実装。
- [ ] 無料体験（登録から1週間プレミアム無料など）の設計。
- [ ] カスタム背景・スリーブアップロード機能の解放（プレミアム限定）。
- [ ] 各種大会結果をまとめているブロガーへのDM営業。「ツールを使って画像出力しませんか？」
- [ ] UIの継続的改善（バグ修正、スマホでの誤タップ防止改善など）。
