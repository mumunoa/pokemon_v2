# Phase 2: 推奨技術スタックと実装・公開ロードマップ（コスト最小化構成）

ご提示いただいたツール群（Supabase, Vercel, Cloudflare, Clerk, Stripe, Resend, Figma, Namecheap, GitHub, Upstash, Pinecone, PostHog, Sentry）は、**現在のNext.js（App Router）環境において、世界中で最も成功している王道の「モダン・サーバーレスSaaSスタック」**です。

結論から申し上げると、**これら全てのツールを統合することは完全に可能であり、かつ初期段階（MVP〜数百ユーザー規模）までは「ほぼ無料（ドメイン代のみ）」で運用できる**という最大のメリットがあります。

以下に、各ツールの役割、全体のアーキテクチャ構成、そして実装からツール公開までの具体的なステップをまとめました。

---

## 1. 各ツールの役割とコスト感（初期費用を抑える構成）

この構成の強みは「従量課金または手厚い無料枠（Free Tier）」を持つサービスだけで構築されている点です。

| カテゴリ | 採用ツール | 本プロジェクトでの役割 | コスト感（初期〜MVP規模） |
| :--- | :--- | :--- | :--- |
| **ホスティング** | **Vercel** | Next.jsアプリのデプロイ、フロントエンド＋API配信 | **無料** (Hobbyプラン。商用化時はPro $20/月) |
| **ドメイン** | **Namecheap** | `your-pokemon-tool.com` などの独自ドメイン取得 | **年額 1,500円前後**（唯一の固定費） |
| **DNS/CDN** | **Cloudflare** | ネームサーバー管理、DDoS保護、キャッシュによる高速化 | **無料** (Freeプランで十分すぎる性能) |
| **バージョン管理** | **GitHub** | ソースコード管理と、Vercelへの自動デプロイ連携 | **無料** |
| **デザイン** | **Figma** | UIデザインの作成、アセット（自作カード枠など）の作成 | **無料** (Starterプラン) |

## 2. システムアーキテクチャ構成図

フロントエンドからバックエンド、インフラまでがシームレスに連携する構成になります。

```text
[ユーザー (ブラウザ/スマホ)]
       │
       │ (DNS解決 & キャッシュ)
   [Cloudflare]
       │
       ▼
 [Vercel (Next.js)] ────── (エラー監視) ─────> [Sentry]
   │           │    ────── (行動分析) ───────> [PostHog]
   │ Frontend  │
   │           └─── [Clerk] (認証・ログイン)
   │
   │ API Routes (Server Actions)
   │
   ├──> [Supabase] (PostgreSQL DB: ログ・デッキ保存)
   ├──> [Upstash]  (Redis: キャッシュ・連打防止)
   ├──> [Pinecone] (Vector DB: AIエンジン用ナレッジ)
   ├──> [Stripe]   (決済API: サブスクリプションステータス確認)
   └──> [Resend]   (メール送信API)
```

## 3. 実装・公開までの具体的なStep（最速リリース手順）

一気にすべてを連携させると開発がストップしてしまうため、**「最小限の公開（MVP）」から「商用化基盤の完成」まで、6つのStepに分けて**実装を進めるのが確実です。

### Step 1: 基礎環境のセットアップとVercel仮公開（★今すぐ実施可能）
まずは現状のツールをインターネット上で触れるようにします。

1. **GitHub**: 現在のローカルコードをGitHubリポジトリにPushします。
2. **Vercel**: Vercelにログインし、GitHubリポジトリを連携してデプロイします。
   - *この時点で `プロジェクト名.vercel.app` というURLが発行され、スマホ実機や友人に共有可能になります。*
3. **Figma**: 画面の微調整が必要な場合や、後で自作カードの枠デザインを作るためにプロジェクトを作成しておきます。

### Step 2: 本番ドメインの適用と公開基盤の完成（コスト発生：年1500円）
独自ドメインを取り、ちゃんとした「Webサービス」の形にします。

1. **Namecheap**: サービス名に合わせたドメイン（例: `pokestrat.io` 等）を購入します。
2. **Cloudflare**: Cloudflareにアカウントを作り、NamecheapのネームサーバーをCloudflareに向けます。
3. **Vercel連携**: Vercelの独自ドメイン設定画面にCloudflareで管理しているドメインを紐付けます。SSL証明書はVercelが自動で無料発行してくれます。

### Step 3: アカウントシステムとDB保存の導入（ユーザー登録開始）
ユーザーにデッキや一人回しの状態を保存させる機能を作ります。

1. **Clerk**: Clerkでプロジェクトを作成し、Next.jsに導入（`<ClerkProvider>` をラップするだけ）。Googleログインとメールアドレスログインを有効化します。
2. **Supabase**: Supabaseでプロジェクトを作成。`users`, `decks`, `game_logs` などのテーブル（スキーマ）を設計します。
3. **Next.js実装**: ユーザーが「保存」ボタンを押した際、ClerkのユーザーID（`userId`）をキーにして、SupabaseのDBに盤面データをINSERTする処理を書きます。

### Step 4: 運用監視とユーザービリティの向上
公開後にバグが出た際や、ユーザーの動きを知るための基盤を入れます。

1. **Sentry**: Next.jsプロジェクトにSentry SDKをインストール（Vercel連携で一発です）。「スマホでカードがタップできない」等のエラーが自動でSentryに飛ぶようになります。
2. **PostHog**: PostHogを導入し、「どのデッキが多く回されているか」「AI分析ボタンは押されているか」を計測します。
3. **Resend**: ユーザー登録完了時のウェルカムメールなどをResend経由で送信するように設定します。

### Step 5: 高速化とインフラ補強
ユーザーが増えてきた際に対応するための準備です。

1. **Upstash**: 一人回しの「一時保存データ」や、APIの過剰な呼び出し（連打）を防ぐためのRate Limit（制限）としてRedisを導入します。DB（Supabase）の負荷を劇的に下げられます。

### Step 6: 課金システムの統合とAI化（マネタイズ開始）
いよいよ課金プランと、目玉となるAI機能を統合します。

1. **Stripe**: Stripeで「月額980円」などの商品（Product）を作成し、Payment Linkを発行します。
2. **Next.js連携**: StripeのWebhookを受け取り、Supabaseのユーザーテーブルの `is_premium` フラグを `true` に更新します。これでアプリ内で「有料機能のロック解除」ができます。
3. **Pinecone**: ポケカの定石やnoteのテキストデータをベクトル化してPineconeに保存。（Python等の外部スクリプト利用を推奨）
4. **AIコーチ稼働**: ユーザーが課金機能の「AIに聞く」を押した際、Pineconeから類似した強い盤面データを検索し、LLM（OpenAI等）を経由してアドバイスを返答するAPIを構築します。

---

## 4. 総括：どの順序で進めるべきか

現状、**「まずは手元のiPhoneで触れるようにする」**というニーズが最優先ですので、**Step 1（GitHub + Vercel仮公開）** に今すぐ進むべきです。

これだけで、現状の良いUIを一気にテストできるようになります。その後、UIが固まってきたら Step 2 (Namecheap + Cloudflare) → Step 3 (Clerk + Supabase) と、機能を足していくのが最も安全でモチベーションの保てるアプローチです。
