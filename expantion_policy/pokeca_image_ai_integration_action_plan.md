# ポケカ一人回しツール 画像対応＆AIコーチ実装の具体ステップ

ご自身でまとめられた `pokeca_image_architecture_nextjs.md` と `pokeca_ai_coach_full_design.md` の内容を確認しました。これらの設計は、前回の「画像を完全に捨てる」方針から一歩踏み込み、**「法的リスクをブラウザ内に閉じ込める（サーバーで配信しない）」ことで、ユーザーに画像の魅力を提供しつつ安全にマネタイズする**という、極めて現実的かつ優れたアプローチです。

以下に、現在のUIからこのアーキテクチャへ移行し、最終的なAIマネタイズ（プレイAIコーチ）へ繋げるための**具体的な実装ステップ**をまとめました。

---

## 開発フェーズと実装ステップ

### Phase 1: 「画像を持たないコアエンジン」の完成（完了）

**目的: 画像がなくてもツールが成立する状態を作り、システムとUIを完全に分離する。**

1.  **[完了] テキストベースUI（MinimalCard / Compact Mode）への改修**
    *   `Card.tsx` を改修し、`displayMode` ('text' | 'compact' | 'local-image') を導入しました。
    *   画像がない場合でも、カード名、タイプ（色分け）、HPなどの情報をCSSで表現するUIを実装済みです。
2.  **[完了] 公式デッキコード・パーサーの実装**
    *   トレーナーズウェブサイトのデッキコードから、ポケモン、トレーナーズ、エネルギーの全カテゴリーを正確に抽出するパーサーを実装しました。
    *   プレイヤー1・プレイヤー2の2つのデッキを独立して読み込み、対戦を開始できるUIも実装済みです。
3.  **[完了] 状態管理（Zustand）と 1手ログ記録の実装**
    *   盤面（State）の操作を Zustand に完全移行し、Undo/Redo 機能を実装しました。
    *   操作ごとに自動的に `logs`（行動履歴）が保存され、画面右側の「ログ」パネルからいつでも確認できる状態を構築済みです。

*   **完了条件**: 外部画像が一切なくても、デッキコードからデッキが生成され、テキストベースでサクサクと一人回しができ、そのログがブラウザの一時メモリに残る状態。

---

### Phase 2: 「詳細時のみ画像・ローカル画像対応」の後乗せ

**目的: サーバーから画像を配信しない形式で、UX（画像表示）を強化する。**

4.  **詳細モーダル（CardFocusModal）の実装**
    *   盤面のカード（自作カードUI）をタップ・長押しした時のみ表示されるモーダルを作ります。
5.  **画像URL解決層（Image Layer）の切り離し**
    *   ゲームの State（盤面状態）とは別に、「カードIDから画像のURL（またはローカルパス）を解決する」ための関数群（`imageRegistry.ts`）を作ります。
6.  **上級者向けローカル画像インポート機能の実装**
    *   設定画面（`ImageImportPanel`）を作り、ユーザーが自分のPC/スマホ上にある画像をドラッグ＆ドロップまたはファイル選択で読み込めるようにします。
    *   読み込んだ画像はサーバーには送らず、ブラウザの IndexedDB にローカルURL（Blob URL 等）として保存し、各カードIDと紐付けます。

*   **完了条件**: 基本はテキスト＋CSS。カードをタップした詳細表示や、ユーザー自身が画像を登録（設定）した場合に限り、ブラウザ内で画像が表示される状態。
    *(※万が一、公式画像のスクレイピングAPIを利用する場合は、ユーザー個人のブラウザからの直接リクエストに留め、システム側の中央サーバーを介さない形にするなど、自己責任の範囲にする防衛策が必須です。)*

---

### Phase 3: AIコーチ基盤の構築とマネタイズ導線

**目的: AIによる分析価値を提供し、サブスクリプション（課金）に結びつける。**

7.  **1試合の「プレイログ」のエクスポート機能**
    *   Phase 1で作った Zustand の行動履歴を、1ゲーム終了時にJSON形式等で出力（またはサーバーDBへ保存）する口を作ります。
8.  **AI分析エンジンとの接続（モックアップ）**
    *   盤面下部に「AI分析」ボタンを設置します。（ここが課金ポイントになります）
    *   まずは、高度なPythonサーバーを立てる前に、Next.js の API Routes（またはブラウザ側）で「初手の手札から必要なサポートを引けているか？」などの簡易な確率計算を返し、AIの価値を仮で提供します。
9.  **Stripeの導入と「プレイ振り返りレポート」の有料枠化**
    *   無料では「対戦ログの保存（〇件まで）」や「簡易な初手確率」までとします。
    *   有料課金（月額980円等）を行うことで、ログをバックエンドの本格的な「盤面理解＆最適手探索AI（Python等で構築）」に投げ、「あのターンは博士ではなくナンジャモが正解だった」というような深いコーチングレポートを生成・閲覧できる権限を付与します。

    *   `displayMode` による切り替えと自作UIの実装が完了しました。
