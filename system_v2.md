# TCG Practice Tool (ポケモンカード一人回し・対戦シミュレーター) 要件定義書 v2

## 1. 3ヶ月/月10万円達成のためのフィージビリティ評価と方針転換
元の設計（完全なルールエンジンの実装によるゲーム進行の自動化、マルチプレイ）を**3ヶ月の期間で行うのは技術的・リソース的に極めて困難**です。ポケカの複雑なスキル処理やカード追加に対応するための運用コストだけで破綻します。
また、ユーザーが「月300円」を払う理由は「ただ対戦できるから」ではなく、「自分の勝率を上げるための分析ができるから」です。

### 改善案（ピボット戦略）
1. **ルールエンジンの完全撤廃（究極のサンドボックス）**
   システム側での「ワザの処理（複雑な状態異常や細かいカード効果判定など）」や「ターンの厳密な制御」は行わず、手動でカードを動かすUIに特化します。ただし、**「指定した数値のダメージを相手ポケモンに自動で載せる」「HPを超えたら自動でトラッシュへ移動を促す気絶判定」といった単純な計算の自動化はプログラマティックに実装可能**であり、ユーザー負担を軽減しつつ開発工数を圧縮します。
2. **「競技プレイヤー（ガチ勢）」向けに特化**
   「最初の7枚を100回引いて事故率を検証するツール」「特定盤面を保存して一人で何手もシミュレーションするツール」など、対人戦にはない**デジタルだからこそできる分析機能**をプレミアム（月額300円）として提供します。

## 2. 対象プラットフォーム
- フェーズ1（MVP）および今後の主力: **Webブラウザ（PC, タブレット, スマホのレスポンシブ対応）**
- **将来的なモバイル対応**: ネイティブアプリ（iOS/Androidストア配信）は、Apple/Googleの厳しい知財審査（リジェクトリスク）および30%の手数料（Apple税・Google税）を回避するため見送りとし、**PWA（Progressive Web App）** としての提供を最優先とします。
- 推奨技術スタック: Next.js (React) + Supabase。X（Twitter）からの1クリック流入を最大化するためWebに特化。

## 3. マネタイズモデルの再構築
徹底したFreemium（フリーミアム）モデル。
**基本無料**:
- 1デッキのインポート、サンドボックスでの一人回し（機能制限あり）
- 初期手札（7枚ドロー）の簡易シミュレーション

**プレミアムプラン群 (月額 500円〜1,000円帯を想定)**:
- 無制限のデッキセーブ数
- 盤面のステート保存・ロード機能（「このターンの状態」を保存し、別ルートの手順を試すUndo/Redo機能）
- 高度なドローシミュレータ（100万回シミュレーションによる、特定カードを引く確率計算等）
- [Elite機能] 次の手の最適解を提示するAIアドバイザー機能
- ※詳細はビジネスプラン `05_ai_and_roadmap_proposal.md` を参照。
- カスタム背景、カスタムスリーブの設定機能

## 4. 機能要件 (MVP向け)

### 4.1 デッキインポート (変更なし)
ユーザー入力のコードから公式構築ページをスクレイピングしてカード情報のみ取得（公式画像はキャッシュせず、都度取得、またはテキストのみ表示）。

### 4.2 サンドボックスエンジン
- **ルール非依存の完全自由操作**: カードをドラッグ＆ドロップでどこへでも移動可能。
- **ステータスオーバーレイ**: カードに対してHP、ダメージ、エネルギー付与の手動操作が可能。

## 5. UI/UX・デザイン要件
「Pokémon TCG Live (PTCGL)」の本格的な盤面と、「Pokémon Trading Card Game Pocket (ポケポケ)」の直感的なステータス表示の融合。（別紙 `design.md` , `filed.md` および Mockup を参照）

## 6. システムアーキテクチャ
- **フロントエンド**: Vercel + Next.js (Tailwind CSS)
- **バックエンド/DB**: Supabase (PostgreSQL, 認証機能)
- **決済**: Stripe Payment LinksまたはStripe Checkout（実装最速化）
- **状態管理**: Zustand または Redux Toolkitによる盤面ツリー管理

## 7. 拡張性の担保（スケーラビリティ）
初期MVPではサンドボックスと手動操作を前提としますが、盤面の状態（State）はすべてJSON形式のツリー構造（React/Zustand等）として厳密に管理できるアーキテクチャにします。
これにより、ユーザー増加後には蓄積されたログデータを元に「AIによるプレイング評価（最適解の提示）」や「特定定石スクリプトの自動再生（COM対戦の簡易版）」といったリッチ機能群システムを根幹から作り直すことなく、このJSONツリーにAPIでアクセスさせるだけで段階的にアドオンできる設計となっています。
