# 01. 必要な画面一覧 (Screens List)

本アプリ「TCG Practice - Sandbox」における主要な画面の一覧と、それぞれの役割を定義します。

## 1. ホーム画面 (Home / Dashboard)
- **役割**: アプリ起動時に表示されるメインメニュー。デッキの選択、新規作成、および一人回し（サンドボックス）への導線となる。
- **主要な要素**:
  - メニューナビゲーション（ホーム、デッキ構築、設定、アカウント/プロフィール）
  - 最近使用したデッキのリスト（プレイ開始ボタン付き）
  - デッキ「新規作成」ボタン
  - プレイヤーステータス（無料プラン・プレミアムプランの表示など）

## 2. デッキインポート・管理画面 (Deck Importer & Manager)
- **役割**: プレイヤーが公式ホームページ（ポケモンカードゲーム トレーナーズウェブサイト）で作成した「デッキコード」を入力し、本アプリ内にデッキを取り込むための画面。自前で全カードデータベースを持たず、必要な画像とテキストのみをその都度スクレイピングして表示することで、著作権リスクと運用コストを最小化する。
- **主要な要素**:
  - デッキ名の編集・管理機能
  - 公式デッキコードのインポート入力フィールド＆実行ボタン
  - 読み込まれたデッキの採用カードリスト確認（画像・枚数） ※カード検索などの構築機能は持たず、デッキの微調整が必要な場合は公式でコードを再発行させる仕様。
  - デッキの保存（Save）ボタン、および一人回し開始ボタン

## 3. アリーナ画面 (Battle Arena / Sandbox)
- **役割**: メインとなる一人回し用のプレイマット画面。「TCGの盤面」を忠実に再現し、全カードを自由に動かせるサンドボックス環境。
- **主要な要素**:
  - **自分陣地**: アクティブポケモン、ベンチ（5枠）、山札、トラッシュ、サイド（6枚）、手札
  - **相手陣地**: （一人回しにおいては、主に仮想敵の展開や自分フィールドを反転させた状態でのテストに使用）アクティブ、ベンチ、山札、トラッシュ、サイド
  - **共有エリア**: スタジアム枠
  - **操作UI**: 
    - 「視点切り替え」ボタン（180度回転し、相手視点で操作）
    - 各カードに対するタップ/左クリックによるメニュー表示（カードの詳細確認、ダメカン操作、ワザ宣言など）
    - ドラッグ＆ドロップによる自由な位置移動
    - ターン進行（Turn End）や手札のリセット・山札の圧縮などを操作できるツールバーやLogボタン

## 4. アカウント / 設定画面 (Settings & Account)
- **役割**: ユーザーのログイン管理、プランのアップグレード（プレミアム移行）、UIカスタマイズ（スリーブ・プレイマット背景）を行う。
- **主要な要素**:
  - ログイン/ログアウトボタン、アカウント情報表示
  - プレミアムプランへのアップグレード（Stripe決済導線）
  - カスタム背景・スリーブアップロードUI（※プレミアム限定）
  - 音量調整や操作感の設定（ダブルタップでカード回転などショートカット設定）

## 5. 分析・シミュレーション画面 (Analytics / Probability) ※Phase 2以降
- **役割**: デッキの初手確率や、特定カードが引けるドロー確率などを計算・表示する画面。
- **主要な要素**:
  - 算出したい条件の入力（例：「手札7枚にたねポケモンが来る確率」）
  - グラフを用いた視覚的な確率表示
